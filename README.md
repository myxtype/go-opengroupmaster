# GroupMaster Bot (Go + SQLite)

基于 Telegram Bot API 的群管理机器人模板，按 PRD 拆分为 `handler/service/repository` 分层，默认可运行 MVP。

## 已实现 MVP

- 群/用户入库与管理员同步（通过 `getChatAdministrators`）
- 私聊 `/start`、`/groups` + Inline Keyboard 管理流
- 欢迎新成员（支持自定义文案）
- 违禁词检测（命中删除消息）
- 自动回复（关键词匹配）
- 简单积分累计（发言 +1）
- 抽奖流程：`/lottery_create`、关键词参与、`/lottery_draw`

## 运行

1. 复制配置

```bash
cp .env.example .env
```

2. 设置 `BOT_TOKEN`

3. 安装依赖并运行

```bash
go mod tidy
go run ./cmd
```

## 命令示例

群内：

- `/lottery_create 活动抽奖|2`
- 在群内发送抽奖关键词（如：`参加`）
- `/lottery_draw`

私聊机器人：

- `/start`
- `/groups`

## Inline Keyboard 操作流

1. 私聊发送 `/start`
2. 点击 `📊 我的群组`
3. 选择要管理的群
4. 在群面板点击按钮：
   - `⚙️ 设置`：切换语言（中文/英文，逐步覆盖）
   - `📨 邀请链接`：生成邀请链接（支持过期小时和人数上限）
   - `📋 接龙`：创建接龙、追加条目、关闭接龙
   - `🗳 投票`：创建群投票、结束当前投票
   - `👁 关键词监控`：配置命中词，命中后私聊通知管理员
   - `🤖 新增自动回复`：按 `关键词=>回复内容` 发送
   - `📄 自动回复列表`：分页查看，支持按钮编辑/删除
   - `🚫 新增违禁词`：发送单个词
   - `📄 违禁词列表`：分页查看，支持按钮编辑/删除
   - `👋 欢迎开关`：切换欢迎消息启用状态
   - `✍️ 编辑欢迎语`：自定义进群欢迎文案（支持 `{user}` 占位符）
   - `🚫 反垃圾开关`：切换链接检测删除
   - `⚡ 反刷屏开关`：切换刷屏检测（超限禁言 60 秒）
   - `🧩 进群验证开关`：新成员需点击验证按钮，超时移出
   - `🧠 验证类型切换`：按钮验证 / 算术验证切换
   - `🔒 新成员限制开关`：新成员 10 分钟内禁发链接/媒体
   - `⏱ 新人时长切换`：10/30/60 分钟轮换
   - `🧹 系统消息清理`：自动清理群系统消息（可逐项开关）
   - 系统清理支持一键预设：`严格 / 推荐 / 关闭`
   - `⏰ 定时消息`：分页查看、启停、删除
   - `➕ 新建定时`：按 `cron表达式=>消息内容` 创建
   - `📊 数据统计`：查看活跃榜（按发言积分）
   - `📜 管理日志`：查看操作日志（分页、筛选、导出 CSV）
   - `🧭 权限分级`：设置 super_admin/admin 角色 + 功能 ACL
   - `⛔ 黑名单`：全局黑名单（跨群生效）
   - `🎯 创建抽奖`：按 `标题|人数` 发送
   - `🏆 立即开奖`：直接开奖并发到目标群

补充：

- 群列表支持分页（默认每页 6 个群）
- 规则列表支持分页（默认每页 5 条）
- 进入输入态后支持 `取消并返回群面板`
- 定时消息创建示例：`0 9 * * *=>早安，今天也要保持活跃`
- 新增定时任务创建成功后即时注册到调度器，无需重启 Bot
- 每个群最多创建 3 条定时消息（超出会提示失败）
- 按钮导航默认编辑原消息，不再每次新发一条，聊天窗口更干净
- 进群验证回调在群内消息按钮触发，验证成功后恢复发言权限
- 算术验证会生成多选题（Inline 按钮），点错不会通过
- 日志导出为 CSV 文件发送到当前私聊会话
- 欢迎文案支持 `{user}`，可 @新成员；未写占位符时会自动附加新成员信息
- 系统消息清理默认配置：
  - 进群: 启用✅
  - 退群: 启用✅
  - 置顶: 关闭❌
  - 修改头像: 关闭❌
  - 修改名称: 关闭❌

## 注意事项

- SQLite 已启用 WAL：`PRAGMA journal_mode=WAL`
- Telegram 无法直接查询“某用户管理的所有群”，当前采用“机器人入群后同步管理员”方案。
- 生产建议切换 webhook、增加限流/审计日志、并把数据库迁移到 PostgreSQL。
