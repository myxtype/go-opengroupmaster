# OpenGroupMaster

[![Go Version](https://img.shields.io/badge/Go-1.26-00ADD8?logo=go)](https://go.dev/)
[![Database](https://img.shields.io/badge/DB-SQLite%20%7C%20PostgreSQL-blue)](https://www.sqlite.org/)
[![Telegram](https://img.shields.io/badge/Platform-Telegram-2CA5E0?logo=telegram)](https://telegram.org/)

开源 Telegram 群管理机器人，基于 Go 构建，采用 `handler/service/repository` 分层架构。支持 SQLite 和 PostgreSQL，开箱可跑，适合个人群运营和中小社区自动化管理。

- GitHub: <https://github.com/myxtype/go-opengroupmaster>
- Issue 反馈: <https://github.com/myxtype/go-opengroupmaster/issues>

## 核心特性

- 群/用户入库与管理员同步（通过 `getChatAdministrators`）
- 私聊 `/start`、`/groups` + Inline Keyboard 管理流
- 欢迎新成员（支持自定义文案）
- 违禁词检测（支持状态开关、独立惩罚面板、提醒自动删除）
- 自动回复（关键词匹配）
- 积分系统（签到/发言/邀请积分、抽奖消耗积分、手动加减积分）
- 抽奖流程：`/lottery_create`、关键词参与、`/lottery_draw`
- 反垃圾规则引擎 + 可选 AI 判定（Ollama）

## 环境要求

- Telegram Bot Token（通过 `@BotFather` 获取）
- 数据库：
  - 默认 SQLite（本地文件）
  - 或 PostgreSQL（生产建议）
- 运行方式二选一：
  - 源码运行：需要 Go `1.26`（见 [go.mod](go.mod)）
  - 可执行文件运行：不需要安装 Go

## 直接下载可执行文件（推荐）

如果你不想安装 Go，可以直接下载预编译二进制运行。

1. 打开 Releases 页面下载对应系统文件：

- <https://github.com/myxtype/go-opengroupmaster/releases>

2. 选择与你系统匹配的压缩包（示例）：

- Linux x64：`groupmasterbot-linux-amd64.tar.gz`
- Linux ARM64：`groupmasterbot-linux-arm64.tar.gz`
- macOS Intel：`groupmasterbot-darwin-amd64.tar.gz`
- macOS Apple Silicon：`groupmasterbot-darwin-arm64.tar.gz`
- Windows x64：`groupmasterbot-windows-amd64.exe.tar.gz`

3. 解压并运行：

- Linux/macOS：

```bash
tar -xzf groupmasterbot-linux-amd64.tar.gz
chmod +x groupmasterbot-linux-amd64
./groupmasterbot-linux-amd64
```

- Windows：
  - 解压后双击 `groupmasterbot-windows-amd64.exe`，或在终端执行：

```powershell
.\groupmasterbot-windows-amd64.exe
```

4. 首次运行前准备 `.env`：

```env
BOT_TOKEN=123456:replace-with-your-bot-token
DB_PATH=sqlite://./data/bot.db
```

可选：Release 附带 `checksums.txt`，可用于校验下载文件完整性。

## 快速开始（源码运行）

如果你希望自行编译或参与开发，请使用源码运行方式。

1. 克隆仓库

```bash
git clone https://github.com/myxtype/go-opengroupmaster.git
cd go-opengroupmaster
```

2. 复制配置

```bash
cp .env.example .env
```

3. 编辑 `.env`，至少填写 `BOT_TOKEN`

```env
BOT_TOKEN=123456:replace-with-your-bot-token
DB_PATH=sqlite://./data/bot.db
```

4. 安装依赖并启动

```bash
go mod tidy
go run ./cmd
```

启动后默认使用 SQLite（`./data/bot.db`），首次运行会自动建表。

## 配置说明

| 变量名 | 必填 | 说明 | 默认值 |
| --- | --- | --- | --- |
| `BOT_TOKEN` | 是 | Telegram 机器人 Token | 无 |
| `DB_PATH` | 否 | 数据库连接串（支持 SQLite/PostgreSQL） | `sqlite://./data/bot.db` |
| `GORM_LOG_SILENT` | 否 | 是否关闭 SQL 日志 | `false` |
| `UPDATE_WORKERS` | 否 | Update 并发 worker 数 | `8` |
| `ADMIN_SYNC_INTERVAL_SECS` | 否 | 管理员同步最小间隔秒数 | `300` |
| `ANTI_SPAM_AI_MODEL` | 否 | AI 反垃圾模型名（为空即禁用 AI） | 空 |
| `ANTI_SPAM_AI_SERVER_URL` | 否 | Ollama 服务地址 | `http://127.0.0.1:11434` |
| `ANTI_SPAM_AI_TIMEOUT_SECS` | 否 | 单次 AI 判定超时秒数 | `8` |

数据库连接示例：

- SQLite：`sqlite://./data/bot.db`
- SQLite 内存：`sqlite://:memory:`
- PostgreSQL：`postgres://user:password@127.0.0.1:5432/groupmaster?sslmode=disable`
- 也支持：`postgresql://...`、`pgsql://...`

## 启用 AI 智能反垃圾（可选）

如果你想使用“AI 智能反垃圾”，需要先安装并启动 Ollama，然后配置 `ANTI_SPAM_AI_MODEL`。

### 1. 安装 Ollama

- macOS（Homebrew）：

```bash
brew install ollama
```

- Linux：

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

- Windows：
  - 打开 <https://ollama.com/download> 下载并安装

### 2. 启动 Ollama 服务

```bash
ollama serve
```

### 3. 拉取一个可用模型（示例）

```bash
ollama pull qwen2.5:1.5b
```

### 4. 配置 `.env`

```env
ANTI_SPAM_AI_MODEL=qwen2.5:1.5b
ANTI_SPAM_AI_SERVER_URL=http://127.0.0.1:11434
ANTI_SPAM_AI_TIMEOUT_SECS=8
```

### 5. 重启机器人后生效

- 配置了 `ANTI_SPAM_AI_MODEL`：反垃圾面板会显示“AI 智能反垃圾设置”
- 没配置 `ANTI_SPAM_AI_MODEL`：AI 面板不显示，且不能开启 AI 反垃圾

## 构建与发布

项目已提供构建脚本：

```bash
# 默认构建当前机器可执行文件到 bin/groupmasterbot
./scripts/build.sh

# 指定目标平台
./scripts/build.sh linux-amd64

# 一次构建多个平台产物
./scripts/build.sh all
```

支持目标：`local`（默认）、`linux-amd64`、`linux-arm64`、`darwin-amd64`、`darwin-arm64`、`windows-amd64`、`all`

## 命令示例

群内：

- `/help`（查看群内常用命令）
- `/lottery_create 活动抽奖|2`
- 在群内发送抽奖关键词（如：`参加`）
- `/lottery_draw`
- `/link`（成员生成专属邀请链接并查看个人邀请统计）
- `/black_add @用户名 原因(可选)`（管理员，仅当前群生效）
- `/black_remove @用户名`（管理员，仅当前群生效）
- 也可回复目标消息执行：`/black_add 原因(可选)`、`/black_remove`

私聊机器人：

- `/start`
- `/help`（查看常用命令与说明）
- `/groups`

## Inline Keyboard 操作流

1. 私聊发送 `/start`
2. 点击 `📊 我的群组`
3. 选择要管理的群
4. 在群面板点击按钮：
   - `⚙️ 设置`：切换语言（中文/英文，逐步覆盖）
   - `📨 邀请链接`：邀请面板（状态开关、过期时间、最大邀请数、生成数量限制、导出、清空）
   - 邀请功能开启后，群成员可在群内使用 `/link` 自动生成链接并查看个人邀请统计
   - 邀请防作弊：仅首次进群计为有效邀请，退群后再通过他人链接入群不重复计数
   - `📋 接龙`：创建接龙、设置限制（人数/时间）、关闭接龙、导出数据
   - `🗳 投票`：创建群投票、结束当前投票
   - `👁 关键词监控`：配置命中词，命中后私聊通知管理员
   - `💰 积分系统`：状态开关、签到口令、发言积分规则、邀请积分规则、积分/排行别名、手动加减积分
   - `🤖 新增自动回复`：四步创建
   - 第1步选择触发方式（精准触发 / 包含触发）
   - 第2步输入关键词
   - 第3步输入回复内容（支持换行）
   - 第4步可选设置链接按钮（支持“跳过”）
   - `📄 自动回复列表`：分页查看，支持按钮编辑/删除
   - `🚫 新增违禁词`：发送单个词
   - `📄 违禁词列表`：分页查看，支持按钮编辑/删除
   - 违禁词管理支持：
   - 状态开关：启用 / 关闭
   - 惩罚设置（独立面板）：
   - 警告：可配置“警告 X 次后”执行 禁言 / 踢出 / 踢出+封禁（支持输入禁言/封禁时长）
   - 禁言：支持输入禁言时长（分钟）
   - 踢出
   - 踢出+封禁：支持输入封禁时长（分钟）
   - 仅撤回消息+不惩罚
   - 删除提醒：命中提醒消息支持自动删除（分钟级）
   - `👋 欢迎开关`：切换欢迎消息启用状态
   - `✍️ 编辑欢迎语`：自定义进群欢迎文案（支持 `{user}` 占位符）
   - `🔠 欢迎按钮`：支持多按钮；格式 `按钮文字 - 链接`，同一行两个按钮用 `&&` 分隔
   - `🚫 反垃圾设置`：状态、惩罚、15项检测开关（含“屏蔽联系人分享”）、例外关键词、删除提醒设置（独立子面板）；配置 `ANTI_SPAM_AI_MODEL` 后才显示 AI 智能反垃圾三级面板（AI 开关、AI 判定垃圾分、AI 严格度：低/中/高）
   - 反垃圾删除提醒子面板支持：`10秒`、`30秒`、`60秒`、`5分钟`、`10分钟`、`30分钟`、`1小时`、`6小时`、`12小时`、`不删除`、`不提醒`
   - 反垃圾提醒文案固定为：
   - `XXX 正在发送垃圾消息。`
   - `[AI广告深度学习模型]`
   - 并附带按钮：`管理员解禁`
   - `⚡ 反刷屏设置`：配置状态、触发阈值、惩罚类型、提醒自动删除（触发条数/检测间隔/删除提醒均为下一级面板固定按钮）
   - `🧩 验证设置`：集中配置进群验证
   - 验证状态：启用 / 关闭
   - 验证时间：下一级面板固定按钮（`1/5/10` 分钟）
   - 超时处理：永久禁言 / 踢出
   - 验证方式：按钮 / 算术题 / 图片验证码 / 中文字符验证码 / 中文语音验证码（五选一）
   - `🔒 新成员限制设置`：进入二级面板配置状态与限制时长
   - 新成员限制时长：下一级面板固定按钮（`10/30/60` 分钟）
   - 新成员限制生效期间：新成员不能发送任何消息
   - `🌙 夜间模式`：状态开关、时区设置（如 `+8`）、夜间时段设置（先设置开始小时 `0-23`，再设置结束小时 `0-23`）、处理方式（删除媒体 / 全局禁言）
   - 夜间时段默认：`00:00-08:00`（按配置时区生效）
   - `💰 积分系统`：状态开关、签到规则、发言规则、邀请规则、积分/排行别名、手动加减积分
   - 签到规则：发送签到口令（默认 `签到`）每天 +1
   - 发言规则：每条消息 +1，支持最小字数限制与每日积分上限
   - 邀请规则：每次有效邀请按配置加分（默认 +1），支持每日积分上限
   - 查询规则：发送积分别名（默认 `积分`）查看个人积分；发送排行别名（默认 `积分排行`）查看积分排行
   - 抽奖规则：发送抽奖关键词参与时消耗积分（默认 `1`）
   - `🧹 系统消息清理`：自动清理群系统消息（可逐项开关）
   - 系统清理支持一键预设：`严格 / 推荐 / 关闭`
   - `⏰ 定时消息`：分页查看、启停、删除
   - `➕ 新建定时`：四步创建
   - 第1步输入 cron 表达式（`分钟 小时 日 月 星期`，共5段）
   - 第2步输入消息内容（支持换行）或直接发送媒体（图片/视频/文件/动图，可带文字说明）
   - 第3步可选设置链接按钮（支持“跳过”）
   - 第4步设置是否置顶（置顶 / 不置顶）
   - 已创建任务支持进入编辑面板，编辑项包括：启用/关闭、是否置顶、修改文本、修改媒体、修改按钮、修改 cron
   - `📊 数据统计`：查看活跃榜（按发言积分）
   - `📜 管理日志`：查看操作日志（分页、筛选、导出 CSV）
   - `🧭 权限分级`：设置 super_admin/admin 角色 + 功能 ACL
   - `⛔ 黑名单`：本群黑名单（仅当前群生效）
   - `🎯 创建抽奖`：按 `标题|人数|口令` 发送（口令可省略，默认 `参加`）
   - `🏆 立即开奖`：直接开奖并发到目标群
   - `📚 创建的抽奖记录`：分页查看抽奖记录，可对“未开奖”活动执行取消
   - 抽奖面板展示统计：创建的抽奖次数、已开奖、未开奖、取消

补充：

- 群列表支持分页（默认每页 6 个群）
- 规则列表支持分页（默认每页 5 条）
- 进入输入态后支持 `取消并返回群面板`
- cron 示例：
  - `0 9 * * *`：每天 09:00
  - `*/30 * * * *`：每 30 分钟
  - `0 21 * * 1-5`：工作日 21:00
- 自动回复和定时消息支持链接按钮，格式示例：
  - `官网 - link.com`
  - `电报 - t.me/WeGroupRobot`
  - `官网 - link.com && 电报 - t.me/WeGroupRobot`
- 编辑自动回复时，按钮步骤支持：`跳过`（保持原按钮）/ `关闭`（清空按钮）
- 新增定时任务创建成功后即时注册到调度器，无需重启 Bot
- 每个群最多创建 3 条定时消息（超出会提示失败）
- 定时消息支持文本与媒体（图片/视频/文件/动图），并支持发送后自动置顶
- 定时消息列表支持切换“是否置顶”
- 按钮导航默认编辑原消息，不再每次新发一条，聊天窗口更干净
- 接龙创建流程：
  - 第1步：选择限制方式（不限制 / 限制人数 / 限制时间 / 人数+时间）
  - 第2步（可选）：输入人数上限
  - 第3步（可选）：通过固定按钮选择截止时长（分钟/小时/天）
  - 最后一步：输入接龙规则或介绍
- 接龙创建成功后，机器人会自动在目标群发送接龙公告：
  - 顶部展示：`🐉 接龙`
  - 展示接龙编号、规则/介绍、截止时间（如 `2026-02-28 10:11:40 UTC+8`）与人数限制
  - 附带按钮：`点击参加接龙`
- 群成员点击 `点击参加接龙` 后会跳转私聊机器人，并提示：`👉 请输入您的接龙内容`
- 用户提交后会收到“接龙成功！”提示；如需修改，可回到群里重新点击 `点击参加接龙` 并重新提交，系统会覆盖原内容
- 同一群可同时存在多个进行中的接龙，用户可分别参与多个接龙
- 每当有用户提交/修改接龙，群内接龙公告会自动更新参与列表（编号、用户、内容）
- 接龙管理支持导出 CSV 数据（包含序号、用户 ID、昵称、内容、更新时间）
- 接龙配置与接龙参与数据已改为独立数据表（`chains` / `chain_entries`）存储，不再写入 `GroupSetting.Config`
- 进群验证回调在群内消息按钮触发，验证成功后恢复发言权限
- 验证成功后会自动删除验证消息（按钮、算术题、图片验证码、中文字符验证码、中文语音验证码均适用）
- 算术验证会生成多选题（Inline 按钮），点错不会通过；连续 3 次失败会立即按“超时处理”惩罚
- 验证码模式会发送一张验证码图片，并提供 4 个数字按钮供选择
- 中文字符验证码会发送一张包含中文字符的图片，并提供 4 个中文字符按钮供选择
- 中文语音验证码会发送一段中文数字语音，并提供 4 个数字按钮供选择
- 验证超时后不会发送群提醒消息，直接执行惩罚
- 邀请面板配置项（过期时间/最大邀请人数/生成数量上限）仅影响新生成链接，不会修改历史链接
- 邀请数据支持 CSV 导出与一键清空（清空后可重新累计）
- 积分系统支持在群内通过关键词触发：签到（默认 `签到`）、积分查询（默认 `积分`）、积分排行（默认 `积分排行`）
- 积分系统支持配置：发言每日上限、发言最小字数、邀请奖励、邀请每日上限、签到/积分/排行别名
- 积分系统支持管理员手动加减积分：先输入用户名/用户ID/转发成员消息，再输入积分数值（正整数）
- 参与抽奖会消耗积分（默认每次 `1` 积分），积分不足时会提示
- 夜间模式支持时区输入：`+8`、`-5`、`+8:30`、`UTC+8`；夜间时段支持按顺序设置开始/结束小时（均为 `0-23`）
- 日志导出为 CSV 文件发送到当前私聊会话
- 欢迎文案支持 `{user}`，会使用 `text_mention` 提及新成员（即使无 username 也可点击）；未写占位符时会自动附加新成员信息
- 黑名单/违禁词/新成员限制等提醒会优先使用 `text_mention`（基于用户 ID），避免出现空 `@`
- 关键词监控命中通知中的“用户”会使用 `text_mention`（基于用户 ID）
- 开奖结果中的中奖用户名单会使用 `text_mention`（基于用户 ID），支持无 username 的可点击提及
- `text_mention` 生效前提：机器人已拿到目标用户 `id`（如来自发言、入群事件、回调查询等）
- mention 文本/实体构造逻辑已统一抽取到 `internal/tgmention`，避免 handler 与 service 重复实现
- 欢迎语、反垃圾/反刷屏提醒、抽奖关键词消息与“参与抽奖成功”提示消息的“自动删除”已改为 SQLite 持久化队列 + 单 worker 执行，重启后会继续处理到期任务
- 反垃圾新增“规则 + AI 判定”流程：
  - 先跑规则引擎，强风险直接拦截
  - 规则检测支持联系人分享（`Contact`）拦截开关
  - 未触发规则引擎，进行 AI 二分类（`spam/ham`）
  - AI 严格度支持 `低/中/高` 三档，按配置切换对应提示词
  - AI 仅接收并输出短 JSON：`{"label":"spam|ham","score":0~100,"reason":"短原因"}`
  - AI 不可用、超时或输出异常时，自动回退到规则引擎，不影响机器人稳定性
- AI 反垃圾判定结果支持 SQLite 缓存（键：`chat_id + content_hash`，字段含 `result_json`、`created_at`，TTL 7 天）
- 系统消息清理默认配置：
  - 进群: 启用✅
  - 退群: 启用✅
  - 置顶: 关闭❌
  - 修改头像: 关闭❌
  - 修改名称: 关闭❌

## 注意事项

- 使用 SQLite 时会自动启用 WAL：`PRAGMA journal_mode=WAL`
- 使用 PostgreSQL 时不会执行 SQLite 专属 PRAGMA
- Update 处理已支持分片并发（默认 8 worker），同一 chat/user 仍保持顺序处理，避免私聊输入态错乱
- Telegram 无法直接查询“某用户管理的所有群”，当前采用“机器人入群后同步管理员”方案。
- 生产建议切换 webhook、增加限流/审计日志；数据库可按场景选择 SQLite 或 PostgreSQL。
